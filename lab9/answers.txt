- 3. I ran bubble_sort on a random array and an already sorted array with 20,000 elements. 
I compiled the program with -O1 and -g, then measured branches and branch misses using perf stat.

- 4. For the random array, Perf reported 44,066,102 branch misses.
This is about 22.0% of the 199,990,000 times the if statement was executed.
For the sorted array, Perf reported 157,965 branch misses, which is about 0.079%.
Perf annotate showed that almost all branch misses happeded at the if statement in bubble_sort. 
This matches the expected result.

- 5. With Perf, increasing the array height from 1 to 10000 increased the L1 data-cache load misses from 21,915,222 to 172,655,526.
The LLC cache misses also increased from 498,317 to 16,766,248, the running time increased from 81.41ms to 511.15ms.
This matched my expectations. A larger height means the program accesses memory farther apart, so there are more cache misses.

- 6. Cachegrind showed that with height 1, there were 8,000,000 data reads and 1,000,001 L1 data-cache read misses.
With height 10000, it still had 8,000,000 data reads, but the L1 cache misses increased to 8,000,000. 
This means that almost every read missed the L1 cache.
This matched my expectations. With height 1, the memory accesses are contiguous, so one cache line can be reused several times.
With height 10000, the accesses are far apart in memory, so almost every read needs a new cache line.
